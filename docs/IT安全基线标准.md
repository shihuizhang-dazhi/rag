| 密码最长使用期限 | 180天 | 密码安全策略 |
| 密码最短使用期限 | 1天 | 密码安全策略 |
| 用可还原的加密来储存密码 | 禁用 | 密码安全策略 |
| 复位帐户锁定计数器 | 15分钟 | 帐户锁定策略 |
| 帐户锁定时间 | 15分钟 | 帐户锁定策略 |
| 帐户锁定阀值 | 3次无效登录 | 帐户锁定策略 |
| guest账号 | 禁止 | 禁用guest用户使用 |
| 帐号检查与管理 | 禁用无需使用的帐号 | — |

## Linux操作系统安全基线标准

### 1、设备管理

统一配置系统安全管理工具，预防远程访问攻击或非授权访问，增加主机系统远程管理安全。

| 基线条目 | 标准要求 | 说明 |
| --- | --- | --- |
| 远程管理工具 | 安装SSH（非最高版本，稳定、符合安全要求为准） | OpenSSH为远程管理高安全性工具，可保护管理过程中传输数据的安全，Linux系统默认安装SSH最高版 |
| 访问控制 | 对于配置文件权限设置不能大于644，对于可执行文件不能大于755 | 应启用访问控制功能，依据安全策略控制用户对资源的访问 |
| 访问控制 | 配置/etc/ssh/sshd_config | 只配置系统管理员、应用管理员IP地址可访问SSHD服务 |

### 2、用户账号与口令安全

账号与口令通常是IT系统的安全弱点，如果不当的配置会使系统账号或口令易被猜解或攻击成功，造成系统被非授权访问或使用。同时，由于账号与口令的安全防护策略力度不强，也会成为攻击弱点。

| 基线条目 | 标准要求 | 说明 |
| --- | --- | --- |
| 禁用默认无用用户 | bin、sys、adm、uucp、nuucp、lpd、imnadm、ipsec、ldap、lp、snapp、invscout | 清理系统中的用户，停用不使用的用户，同时，针对需要使用的用户，制作用户列表进行妥善保存 |
| root远程登录 | 禁止 | 禁止root远程登录，通过普通用户登录之后利用su命令转到系统管理员帐号 |
| 口令策略（/etc/login.defs、/etc/ssh/sshd_config） | PASS_MIN_DAYS 7、PASS_WARN_AGE 28、PASS_MIN_LEN 8、PASS_MAX_DAYS 180、PASS_MIN_DAYS 0、PASS_MIN_LEN 8、PASS_WARN_AGE 7、FAIL_DELAY 10、FAILLOG_ENAB yes、SYSLOG_SU_ENAB yes、SYSLOG_SG_ENAB yes、MD5_CRYPT_ENAB yes、MaxAuthTries 5 | 密码修改最小间隔时间 7天；在密码过期之前28天会收到警告；密码最小8位字符；登录密码有效期180天；登录密码最短修改时间；登录密码最小长度8位；登录密码过期提前7天提示修改；登录错误时等待时间10秒；登录错误记录到日志；限定超级用户管理日志时使用；限定超级用户组管理日志时使用；使用md5为密码的加密方法；最大密码尝试失败次数5 |
| FTP用户帐号控制 | ftpusers | 最大密码尝试失败次数3-6 |

### 3、日志与审计

日志为主机系统内部活动行为记录，对日志的审计与保护是非常必要的，因此，基线标准对日志安全控制与管理，提高对各系统日志的保护。

| 基线条目 | 标准要求 | 说明 |
| --- | --- | --- |
| 禁止接受网络日志 | 关闭UDP514 | 禁止服务器接受网络日志 |
| 日志存储 | kern.warning;\*.err;authpri.none @loghost、\*.info;mail.none;authpriv.none;cron.none @loghost、\*.emerg @loghost、local7.\* @loghost | 使用日志服务器接受与存储主机日志 |
| 日志系统配置文件保护 | 文件属性400（管理员账号只读） | 修改日志配置文件（syslog.conf）权限为400 |

### 4、服务优化

Linux系统默认安装后，系统安装大量默认服务，部分服务存在安全弱点，会给主机系统产生风险，为有效对Linux系统服务进行管理，管理员应根据需求对默认服务进行优化，一方面提高系统安全性，另一方面优化系统资源使用率。

| 基线条目 | 标准要求 | 说明 |
| --- | --- | --- |
| telnet 服务 | 禁止 | 远程访问服务 |
| ftp 服务 | 禁止 | 文件上传服务（需要经过批准才启用） |
| sendmail 服务 | 禁止 | 邮件服务 |
| klogin 服务 | 禁止 | Kerberos 登录，如果站点使用 Kerberos 认证则启用（需要经过批准才启用） |
| kshell 服务 | 禁止 | Kerberos shell，如果站点使用 Kerberos 认证则启用（需要经过批准才启用） |
| ntalk 服务 | 禁止 | new talk |
| tftp 服务 | 禁止 | 以 root 用户身份运行并且可能危及安全 |
| imap 服务 | 禁止 | 邮件服务 |
| pop3服务 | 禁止 | 邮件服务 |
| GUI服务 | 启用 | 图形管理服务 |
| X windows服务 | 启用 | 通用的windows界面 |
| services 启动服务 | 禁止 | 系统自动启动服务：nfs、nfslock、autofs、ypbind、ypserv、yppasswdd、portmap、smb、netfs、lpd、apache、httpd、tux、snmpd、named、postgresql、mysqld、webmin、kudzu、squid、cups、ip6tables、iptables、pcmcia、bluetooth、NSResponder、apmd、avahi-daemon、canna、cups-config-daemon、FreeWnn、gpm、hidd等（部分应用根据实际需要审批开通） |
| 135、137、138、139和445端口 | 禁止 | 禁用135、137、138、139和445端口 |

### 5、安全防护

通过对Linux系统配置调整，提高系统安全。

| 基线条目 | 标准要求 | 说明 |
| --- | --- | --- |
| banner修改 | 修改issue motd | banner信息泄漏Linux系统及服务版本信息，如系统版本信息 |
| Umask权限 | 027 | 修改默认文件权限 |
| 敏感文件安全保护 | chown root:root passwd shadow group、chmod 644 passwd group、chmod 400 shadow | 保护口令文件 |
| 控制root操作历史记录 | 100 | 至少记录100个历史命令，设置其$HOME目录变量下.profile文件属性，确认histsize指定 |
| 运行杀毒软件 | 服务器必须安装并运行统一的企业版杀毒软件，并保持病毒库及时更新 | 必须安装指定的杀毒软件并保证杀毒软件正常运行 |

### 6、其他内容

| 基线条目 | 标准要求 | 说明 |
| --- | --- | --- |
| 安全审计 | 启用审计功能 | — |
| 安全审计 | 审计记录包括事件发生的日期和时间、触发事件的主体与客体、事件的类型、事件成功或失败、身份鉴别事件中请求的来源、事件的结果等 | 审计记录应包括事件的日期、时间、类型、主体标识、客体标识和结果等 |

| 入侵防范 | 遵循最小安装原则，仅安装需要的组件和应用程序；未启动多余的服务和端口；设置升级服务器实现对服务器的补丁升级；操作系统和数据库系统补丁为厂商新公布的补丁版本 | 操作系统遵循最小安装的原则，仅安装需要的组件和应用程序，并通过设置升级服务器等方式保持系统补丁及时得到更新 |
| 资源控制 | 远程登录限制终端接入方式和网络地址 | 应通过设定终端接入方式、网络地址范围等条件限制终端登录 |
| 资源控制 | 设置了合理的操作超时锁定（10分钟以内）；在恢复时需要重新鉴别 | 应根据安全策略设置登录终端的操作超时锁定 |
| 资源控制 | 通过监控管理软件对服务器的资源（CPU、硬盘、内存、网络等）的使用情况进行实时监视 | 应对重要服务器进行监视，包括监视服务器的CPU、硬盘、内存、网络等资源的使用情况 |
| 资源控制 | 通过第三方监控管理软件进行监视；第三方监控管理软件具有报警功能，且设置了报警门限值 | 应能够对系统的服务水平降低到预先规定的最小值进行检测和报警 |

## Oracle数据库安全基线标准

### 1、用户账号与口令安全

账号与口令通常是IT系统的安全弱点，如果不当的配置会使系统账号或口令易被猜解或攻击成功，造成系统被非授权访问或使用。同时，由于账号与口令的安全防护策略力度不强，也会成为攻击弱点。

| 基线条目 | 标准要求 | 说明 |
| --- | --- | --- |
| oracle帐号 | 删除、锁定无用帐号 | 清理帐号，无用帐号删除或锁定 |
| 数据库SYSDBA帐号 | 禁止远程登录 | 修改配置参数，禁止SYSDBA远程登录（Oracle ADG双机除外） |
| 口令策略 | PASSWORD_VERIFY_FUNCTION 8、PASSWORD_REUSE_MAX 5 | 密码复杂度8个字符；禁止使用最近5次使用的口令 |
| public权限 | 优化 | 清理public各种默认权限 |
| 数据库其他账号 | 禁止使用DBA用户，禁止使用Unlimited tablespace及resource权限 | 除特殊应用（如某企业财务系统等对用户权限有特殊要求）外，其他应用的用户应权限最小化 |
| 账号策略 | 登录失败尝试次数配置、账号锁定时间配置 | 根据实际情况，失败尝试次数5-10次，超过次数则锁定账户；根据实际情况，帐号锁定时间12-24小时 |

### 2、日志与审计

日志为主机系统内部活动行为记录，对日志的审计与保护是非常必要的，因此，基线标准对日志安全控制与管理，提高对各系统日志的保护。

| 基线条目 | 标准要求 | 说明 |
| --- | --- | --- |
| 登录日志记录 | 启动 | 针对需要的系统添加数据库审计，对登录日志进行记录 |
| 数据库操作日志 | 启动 | 针对需要的系统添加数据库审计，对操作日志进行记录 |
| 日志文件保护 | 启用 | 多重日志文件组 |

### 3、服务优化

| 基线条目 | 标准要求 | 说明 |
| --- | --- | --- |
| 数据库服务 | 启动 | 仅开放需要的服务，其余全部关闭 |

### 4、安全防护

通过对系统配置调整，提高系统安全。

| 基线条目 | 标准要求 | 说明 |
| --- | --- | --- |
| 数据字典保护 | O7_DICTIONARY_ACCESSIBILITY = FALSE | — |
| 控制文件保护 | 分区备份 | — |
| 日志归档 | 启用 | 新建生产数据库开启归档模式 |

### 5、数据库版本

选择稳定版本，建议使用19c的最新补丁版本，Oracle 11g的版本要使用11.2.0.4，不建议使用12c（12c为中间版本）。

| 基线条目 | 标准要求 | 说明 |
| --- | --- | --- |
| 数据库版本 | 11g使用11.2.0.4；19c使用19.13.0.0 | 使用最新的补丁包版本，确保数据库安全稳定；无特殊情况，不建议使用11g之前的版本作为正式库；无特殊情况，不建议使用Windows服务器作为数据库服务器 |

## MySQL数据库安全基线标准

### 1、用户账号与口令安全

账号与口令通常是IT系统的安全弱点，如果不当的配置会使系统账号或口令易被猜解或攻击成功，造成系统被非授权访问或使用。同时，由于账号与口令的安全防护策略力度不强，也会成为攻击弱点。

| 基线条目 | 标准要求 | 说明 |
| --- | --- | --- |
| 账号 | 口令由数字、大小写字母、符号混排，无规律的方式；用户口令的长度至少为8位，更新的口令至少5次内不能重复 | 应用禁止使用数据库root账号作为业务账号 |
| 用户名 | 数据库系统的用户名具有唯一性 | 为操作系统和数据库的不同用户分配不同的用户名，确保用户名具有唯一性 |
| 登录失败处理 | 已启用登录失败处理功能；限制非法登录尝试次数，超尝试次数后实现锁定策略；设置网络连接超时自动退出 | 应启用登录失败处理功能，可采取结束会话、限制非法登录次数和自动退出等措施 |

### 2、日志与审计

日志为主机系统内部活动行为记录，对日志的审计与保护是非常必要的，因此，基线标准对日志安全控制与管理，提高对各系统日志的保护。

| 基线条目 | 标准要求 | 说明 |
| --- | --- | --- |
| 登录日志记录 | 启动 | 针对需要的系统添加数据库审计，对登录日志进行记录 |
| 数据库操作日志 | 启动 | 针对需要的系统添加数据库审计，对操作日志进行记录 |
| 安全审计 | 采取措施对审计记录进行保护，记录至少保存半年 | 应保护审计记录，避免受到未预期的删除、修改或覆盖等 |

### 3、服务优化

| 基线条目 | 标准要求 | 说明 |
| --- | --- | --- |
| 优化功能 | 性能优化，管理员可以对用户的数据库程序提出优化建议，包括慢SQL的修改建议等 | — |

### 4、其他内容

## 应用系统安全基线标准

### 1、应用管理

避免非授权访问URL。

| 基线条目 | 标准要求 | 说明 |
| --- | --- | --- |
| 管理界面限制 | 互联网隐藏管理界面 | 对互联网发布的应用原则上隐藏管理界面，管理界面仅对内网开放，限制管理界面的登陆地址 |

### 2、用户账号与口令安全

账号与口令通常是IT系统的安全弱点，如果不当的配置会使系统账号或口令易被猜解或攻击成功，造成系统被非授权访问或使用。同时，由于账号与口令的安全防护策略力度不强，也会成为攻击弱点。

| 基线条目 | 标准要求 | 说明 |
| --- | --- | --- |
| 口令策略 | 强制用户首次登录时修改默认口令，或是使用用户自定义初始密码的策略；连续数字或字母、重复数字或字母、键盘上连续排列的数字或字母（如11111、1、qwer、qazwsx、666888）不得使用；不得使用常见密码"password"、"root"、"admin"、"P@ssw0rd"等；禁止使用公司名称或公司名称缩写；完善密码策略，信息安全最佳实践的密码策略为8位（包括）以上字符，包含数字、大小写字母、特殊字符中的至少3种；增加人机验证机制，限制IP访问次数 | 加强口令设置，账户包括管理员、业务人员的账户；用户首次登录应设置强制修改初始口令功能 |
| 账号策略 | 登录失败尝试次数配置、账号锁定时间配置；系统上线或验收前删除测试账号、角色 | 根据实际情况，失败尝试次数3-10次，超过次数则锁定账户；根据实际情况，帐号锁定时间15-60分钟；在系统上线或验收前，应删除开发期间建立的各类测试账号、角色及对应的权限 |
| 帐户密码有效时间 | 不多于180天 | 不多于180天，需强制提示修改密码 |
| 账号实名制 | 每个账号在后台需有至少一个字段能够证明账号使用人员的唯一性，包括姓名、身份证号、工号等 | 管理员、业务人员的账号均需实名，并由业务使用部门发流程申请，无法实名的，请在流程详细说明 |

### 3、日志与审计

日志为主机系统内部活动行为记录，对日志的审计与保护是非常必要的，因此，基线标准对日志安全控制与管理，提高对各系统日志的保护。

| 基线条目 | 标准要求 | 说明 |
| --- | --- | --- |
| 应用系统的日志记录 | 确定日志名称及存储位置 | 通过定义日志文件名设置日志记录与存储 |
| 日志保留时间 | 6个月 | 日志需保存6个月 |
| 日志定期备份 | 1个月 | 每个月对日志进行备份 |

### 4、服务优化

应用系统需要定期针对应用系统进行安全测试。

| 基线条目 | 标准要求 | 说明 |
| --- | --- | --- |
| 新应用系统上线前测试 | 上线前进行漏洞扫描 | 新业务上线前需要进行漏洞扫描检测，无中高风险漏洞，才能上线，要提供漏洞扫描报告 |
| 应用系统例行安全测试 | 定期进行漏洞扫描，一般一年两次 | 例行漏洞扫描一般一年两次，如果应用系统有变更，变更后可申请进行漏洞扫描 |

### 5、安全防护

通过对应用系统配置调整，提高系统安全。

| 基线条目 | 标准要求 | 说明 |
| --- | --- | --- |
| 用户密码文件保护 | 用户账户信息、密码信息密文存储 | 所有业务系统的账户信息和密码都要加密存储，不能明文存储 |
| 连接会话超时控制 | 3-10分钟 | 需要设置业务人员、管理员，3-10分钟无操作即退出会话 |
| 数据传输安全 | SSL证书 | 应用系统要采用正规有效的加密证书，提供HTTPS访问方式 |
| 超级管理员数量 | 超级管理员数量不能超过2个 | 超级管理员数量最多2个，且必须实名 |
