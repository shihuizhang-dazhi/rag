# OWASP Top 10: A05 - 安全配置错误 (Security Misconfiguration)

## 概述

安全配置错误是最常见的安全问题之一。包括不安全的默认配置、不完整的临时配置、开放的云存储、错误的 HTTP 头配置以及包含敏感信息的详细错误消息。应用栈的任何层面都可能出现配置错误。

## 常见场景

### 1. 默认账号密码未修改
- 数据库 root/toor
- Tomcat admin/admin
- Jenkins 未设置密码
- Redis 无密码对外开放

### 2. 不必要的功能启用
- 目录列表（Directory Listing）开启
- DEBUG 模式在生产环境启用
- 不必要 HTTP 方法（PUT、DELETE、TRACE）
- WebDAV 扩展启用

### 3. 错误信息泄露
- SQL 错误回显数据库结构
- 堆栈跟踪暴露代码路径
- 版本号在响应头中暴露

### 4. 云服务配置错误
- S3 存储桶公开访问
- 安全组对外开放过多端口
- IAM 策略权限过大

### 5. HTTP 安全头缺失
- 未设置 CSP (Content-Security-Policy)
- 未设置 X-Frame-Options
- 未设置 X-Content-Type-Options
- 未设置 Referrer-Policy

## 检测方法

```bash
# 扫描开放端口
nmap -sV target.com

# 检查 HTTP 安全头
curl -I https://target.com

# 检测目录列表
curl https://target.com/uploads/

# 扫描默认路径
dirb https://target.com /usr/share/wordlists/dirb/common.txt

# Docker 安全扫描
docker scan image:tag

# S3 公开桶检查
aws s3 ls s3://bucket-name --no-sign-request
```

## 防御措施

- **硬化基线**：参考 CIS Benchmarks 进行系统加固
- **最小配置**：移除不必要的功能、组件、文档和示例
- **自动化加固**：使用 Ansible、Puppet 等工具实现配置自动化
- **安全头配置**：
  ```
  Content-Security-Policy: default-src 'self'
  X-Frame-Options: DENY
  X-Content-Type-Options: nosniff
  Strict-Transport-Security: max-age=31536000
  ```
- **持续监控**：定期审计配置变更，使用云安全态势管理（CSPM）
- **分段部署**：不同环境使用隔离的配置和凭据
- **补丁管理**：及时更新操作系统、中间件、依赖库

## 安全配置检查表

| 检查项 | 要求 |
|--------|------|
| 默认密码 | 全部修改 |
| 目录列表 | 关闭 |
| DEBUG 模式 | 生产环境关闭 |
| HTTP 方法 | 仅允许必要方法 |
| 错误页面 | 自定义、不暴露细节 |
| 安全头 | 全部配置 |
| 端口 | 仅开放必要端口 |
| 云存储 | 默认私有 |
