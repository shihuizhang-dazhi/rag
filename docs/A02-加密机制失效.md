# OWASP Top 10: A02 - 加密机制失效 (Cryptographic Failures)

## 概述

加密机制失效（原称敏感数据泄露）指与密码学相关的安全缺陷，常导致敏感数据泄露或系统被攻破。包括传输和存储过程中的加密问题。

## 常见漏洞场景

### 1. 明文传输
- 使用 HTTP 而非 HTTPS 传输数据
- 敏感信息在网络中被嗅探

### 2. 弱加密算法
- 使用 MD5、SHA1 做密码哈希
- 使用 DES、RC4 等已被破解的加密算法
- 使用 ECB 模式的 AES

### 3. 硬编码密钥
- 密钥、API Key 直接写在代码中
- 提交到 Git 仓库导致泄露

### 4. 未强制 HTTPS
- 支持 HTTP 访问
- 未设置 HSTS 头
- Cookie 未设置 Secure 标志

### 5. 随机数生成不足
- 使用不安全的随机数生成器
- 导致令牌可预测

## 检测方法

```bash
# 检查 SSL/TLS 配置
nmap --script ssl-enum-ciphers -p 443 target.com

# 检测 HSTS
curl -I https://target.com | grep Strict-Transport-Security

# 检查 Cookie 安全标志
curl -I https://target.com | grep Set-Cookie
```

## 防御措施

- **强制 HTTPS**：全站启用 TLS 1.2+，配置 HSTS
- **强加密算法**：密码哈希用 bcrypt/argon2，对称加密用 AES-256-GCM
- **密钥管理**：使用密钥管理服务（KMS），禁止硬编码
- **安全 Cookie**：设置 Secure、HttpOnly、SameSite 属性
- **禁用缓存**：敏感页面设置 `Cache-Control: no-store`
- **证书管理**：定期更新证书，使用自动化工具监控过期

## 密码学最佳实践

| 场景 | 推荐方案 | 禁止方案 |
|------|---------|----------|
| 密码存储 | bcrypt/argon2 | MD5/SHA1/SHA256 |
| 对称加密 | AES-256-GCM | DES/3DES/RC4 |
| 非对称加密 | RSA 2048+ / ECC | RSA 1024 |
| 哈希 | SHA-256/SHA-3 | MD5/SHA1 |
| 随机数 | crypto.randomBytes | Math.random() |
