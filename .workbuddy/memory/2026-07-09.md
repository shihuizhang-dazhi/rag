## RAG 后端实现
- 按截图目标架构新增 `app/backend/db/`（models.py + session.py）和 `app/backend/service/`（chat_service.py + document_service.py + vectorization_service.py）。
- 更新 `config.py`：将 `data/*` 相对路径解析为绝对路径；新增 `chroma_collection_name` 与 `system_prompt`。
- Document 模型：包含原始文件名、文件大小、MIME 类型、存储路径、向量化状态、创建时间；提供 `to_dict()`。
- 数据库会话：SQLite + SQLAlchemy，`get_db()` 供 FastAPI Depends，`db_session()` 供上下文管理。
- 文档服务：upload 校验扩展名/大小、同名去重、UUID 落盘、入库、调用向量化；delete 同时清理磁盘/DB/向量索引；list 支持分页+文件名模糊搜索。
- 向量化服务：使用 `DashScopeEmbeddings`（OpenAIEmbeddings 在 DashScope 兼容端点上报 `input.contents` 错误），按扩展名用 PyPDFLoader/CSVLoader/TextLoader，RecursiveCharacterTextSplitter 分块，存入 Chroma。
- 对话服务：/chat 先从 Chroma 检索，将 Chroma 返回的 distance 转为 similarity（1 - distance）再与 `score_threshold` 比较，命中后注入 system prompt，SSE 返回 content + sources 帧。
- 重写 `main.py`：启动时创建数据目录并 `init_db()`；注册 `GET /documents`、`POST /documents/upload`、`DELETE /documents/{doc_id}`；/chat 接入 chat_service。
- 测试：uvicorn 以 `--reload` 在 8080 运行；上传 rag_demo.txt 后 `is_vectorized=true`；列表、删除正常；POST /chat 返回基于参考资料的答案和 sources 帧，与前端契约对齐。
