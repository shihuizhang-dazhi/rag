# OWASP Top 10: A04 - 不安全设计 (Insecure Design)

## 概述

不安全设计是 OWASP Top 10 2021 新增类别，关注设计与架构缺陷。它与实现缺陷不同——即使代码完美实现，如果设计本身存在缺陷，安全也无法保障。需要在开发早期进行威胁建模和安全设计审查。

## 常见场景

### 1. 缺少输入验证设计
- 系统设计时未考虑输入边界
- 未定义合法的输入范围和格式
- 对超长输入、特殊字符无处理策略

### 2. 不安全的业务流程
- 密码重置流程绕过验证
- 支付金额可被客户端修改
- 优惠券可被重复使用

### 3. 过度暴露
- 错误信息暴露数据库结构
- API 返回过多字段
- 调试接口未在生产环境关闭

### 4. 缺乏速率限制
- 登录接口无限尝试
- API 接口无调用频率限制
- 短信验证码接口未防护

### 5. 不安全的文件处理
- 允许上传任意类型文件
- 上传文件未重命名
- 未限制文件大小

## 威胁建模方法

### STRIDE 模型
| 威胁类型 | 描述 | 对应安全属性 |
|---------|------|------------|
| Spoofing | 身份欺骗 | 认证 |
| Tampering | 数据篡改 | 完整性 |
| Repudiation | 抵赖 | 不可否认性 |
| Information Disclosure | 信息泄露 | 机密性 |
| Denial of Service | 拒绝服务 | 可用性 |
| Elevation of Privilege | 权限提升 | 授权 |

### 安全设计检查清单

- [ ] 是否完成威胁建模文档
- [ ] 敏感数据是否加密存储和传输
- [ ] 是否实施最小权限原则
- [ ] 是否存在默认账号和密码
- [ ] 是否设计了审计日志
- [ ] 错误处理是否屏蔽内部细节
- [ ] 文件上传是否有类型、大小、数量限制
- [ ] API 是否有限流机制
- [ ] 密码策略是否满足复杂度要求
- [ ] 会话管理是否安全（超时、单点登录控制）

## 防御措施

- **安全开发生命周期 (SDL)**：从需求阶段融入安全
- **威胁建模**：使用 STRIDE 等方法识别威胁
- **安全设计模式**：参考 OWASP Cheat Sheet 系列
- **安全需求文档**：明确非功能性安全需求
- **架构评审**：上线前由安全团队评审架构设计
- **确定安全边界**：明确信任边界和数据流向
