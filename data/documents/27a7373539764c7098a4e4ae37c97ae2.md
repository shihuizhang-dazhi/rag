# OWASP Top 10: A08 - 软件和数据完整性故障 (Software and Data Integrity Failures)

## 概述

软件和数据完整性故障是 OWASP Top 10 2021 新增类别。包括不验证软件更新、关键数据和 CI/CD 管道的完整性的情况。攻击者可以篡改软件供应链或数据，导致系统性安全危害。

## 常见场景

### 1. 不安全的反序列化
```python
# 危险的 pickle 反序列化
import pickle
data = request.get_data()
obj = pickle.loads(data)  # 可导致任意代码执行

# Java 反序列化漏洞
ObjectInputStream ois = new ObjectInputStream(inputStream);
MyObject obj = (MyObject) ois.readObject();  // 可被利用
```

### 2. 软件供应链攻击
- SolarWinds 事件：官方更新包被植入后门
- npm 包 typosquatting（冒充相似包名）
- 依赖混淆攻击（利用私有包名冲突）
- 恶意代码注入开源项目

### 3. CI/CD 管道攻击
- 泄露 CI/CD 密钥（GitHub Actions Secrets）
- 构建脚本被篡改
- Docker 基础镜像被投毒
- 制品签名未验证

### 4. 未验证更新
- 自动更新未验证签名
- 使用 HTTP 下载更新包
- 更新镜像未校验哈希值

### 5. 关键数据篡改
- Cookie 值可被客户端修改
- JWT 使用 None 算法绕过
- 序列化对象中包含敏感数据
- 数据库直接修改绕过业务逻辑

## 检测方法

```bash
# 反序列化漏洞探测
# 观察应用是否接收序列化对象
# 测试修改序列化数据后的响应

# 依赖混淆测试
npm install --registry https://registry.npmjs.org <private-package-name>

# Docker 镜像扫描
docker scout quickview image:tag
cosign verify image:tag

# 制品签名验证
gpg --verify artifact.tar.gz.sig artifact.tar.gz
```

## 防御措施

- **数字签名**：对所有软件更新、制品进行签名验证
- **避免不安全反序列化**：
  - 尽量不使用原生序列化
  - 使用 JSON/XML 替代
  - 如必须使用，实施类型白名单
- **软件供应链安全**：
  - 使用依赖锁定文件（package-lock.json）
  - 扫描第三方依赖
  - 验证包来源和完整性
  - 私有包使用 scope 命名空间
- **CI/CD 安全**：
  - 密钥使用专用 Secrets 管理
  - 构建过程隔离
  - 制品签名并校验
  - 最小权限授予 CI/CD 服务账号
- **数据完整性保护**：
  - 关键数据使用 HMAC 签名
  - JWT 验证签名和算法
  - 敏感数据不在客户端存储

## SolarWinds 事件教训

2020 年底爆发的 SolarWinds 供应链攻击是历史上最严重的软件供应链攻击之一。攻击者入侵 SolarWinds 构建系统，在 Orion 平台的官方更新包中植入后门，影响超过 18,000 家客户，包括美国政府机构和财富 500 强企业。

关键教训：
1. 构建管道本身是重要攻击面
2. 即使官方更新包也不可完全信任
3. 需要多层次的软件完整性验证
4. 网络分段和最小权限可限制攻击范围
