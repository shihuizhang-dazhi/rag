# OWASP Top 10: A06 - 脆弱和过时的组件 (Vulnerable and Outdated Components)

## 概述

使用存在已知漏洞的组件（库、框架、模块、操作系统）是普遍且危险的问题。组件通常以与应用程序相同的权限运行，因此利用组件漏洞可以导致严重的数据丢失或服务器接管。

## 常见场景

### 1. 已知漏洞未修复
- Log4Shell (CVE-2021-44228) - Log4j 远程代码执行
- Spring4Shell (CVE-2022-22965) - Spring Framework RCE
- Heartbleed (CVE-2014-0160) - OpenSSL 信息泄露
- Ghostcat (CVE-2020-1938) - Tomcat AJP 文件读取

### 2. 不再维护的组件
- 使用已 EOL 的框架版本
- 使用停止维护的开源库
- 无人维护的 WordPress 插件

### 3. 未知来源的组件
- 使用未经验证的第三方库
- 从非官方源下载依赖
- 供应链投毒攻击

### 4. 未扫描依赖
- 不知道应用中使用了哪些组件
- 未建立软件物料清单 (SBOM)
- 不知道组件的授权协议

## 检测方法

```bash
# Node.js 依赖扫描
npm audit
npm audit fix

# Python 依赖扫描
pip-audit
safety check

# Java/Maven
mvn dependency-check:check

# Docker 镜像扫描
trivy image myapp:latest

# 通用 SCA 工具
snyk test
```

## 典型漏洞案例

| CVE | 影响组件 | 危害 | CVSS |
|-----|---------|------|------|
| CVE-2021-44228 | Log4j 2.x | RCE | 10.0 |
| CVE-2022-22965 | Spring Framework | RCE | 9.8 |
| CVE-2017-5638 | Struts2 | RCE | 10.0 |
| CVE-2019-0230 | Struts2 | RCE | 9.8 |
| CVE-2021-21315 | systeminformation | 命令注入 | 7.8 |

## 防御措施

- **持续监控**：使用 Dependabot、Renovate 自动检测和更新依赖
- **SBOM 管理**：使用 CycloneDX 或 SPDX 维护软件物料清单
- **定期扫描**：集成 SCA（软件组成分析）工具到 CI/CD 流程
- **及时更新**：建立补丁管理流程，定期升级组件
- **最小依赖**：移除不必要的依赖，减少攻击面
- **来源验证**：仅从官方仓库获取组件，验证签名
- **虚拟补丁**：对于无法立即修复的漏洞，使用 WAF 虚拟补丁
- **安全公告订阅**：关注 CVE/NVD、GitHub Advisory、供应商安全公告

## CI/CD 集成示例

```yaml
# GitHub Actions
- name: Run Snyk
  uses: snyk/actions/node@master
  env:
    SNYK_TOKEN: ${{ secrets.SNYK_TOKEN }}
  with:
    args: --severity-threshold=high
```
