# OWASP Top 10: A07 - 身份识别和认证失败 (Identification and Authentication Failures)

## 概述

身份识别和认证失败（原称失效的身份认证）指与用户身份验证和会话管理相关的安全缺陷。攻击者可以利用这些漏洞冒充其他用户、绕过登录或劫持会话。

## 常见漏洞场景

### 1. 弱密码策略
- 允许简单密码（123456、password）
- 无密码复杂度要求
- 不限制密码尝试次数（暴力破解）

### 2. 凭证泄露
- 密码明文存储
- 使用弱哈希算法（MD5、SHA1）
- 通过 GET 请求传递密码

### 3. 会话管理缺陷
- Token 存储在 localStorage（XSS 可窃取）
- Session ID 通过 URL 传递
- 登出后 Session 未失效
- 会话超时时间过长或永不过期

### 4. 认证逻辑缺陷
- 登录失败和成功返回不同错误信息（用户名枚举）
- 密码重置令牌可预测
- 多因素认证可绕过
- 注册时自动分配可预测的用户 ID

### 5. 不使用多因素认证
- 仅依赖用户名密码
- 未提供 TOTP、短信验证码等二次验证
- 关键操作无额外认证

## 攻击方法

```bash
# 暴力破解
hydra -l admin -P passwords.txt target.com http-post-form "/login:username=^USER^&password=^PASS^:Login failed"

# 用户名枚举
curl -X POST https://target.com/login -d "username=admin&password=x"
# 观察不同用户名的响应差异

# JWT 攻击
# 尝试 None 算法
# 尝试弱密钥暴力破解
# 查看 jwt.io 解码信息

# Cookie 会话探测
curl -c cookies.txt https://target.com/login
curl -b cookies.txt https://target.com/dashboard
```

## 防御措施

- **多因素认证（MFA）**：强制启用 TOTP 或硬件令牌
- **密码策略**：
  - 最小长度 8 位以上
  - 包含大小写字母、数字、特殊字符
  - 禁止常见密码和泄露密码
- **安全存储**：密码使用 bcrypt、argon2 哈希，不可逆
- **会话安全**：
  - Token 使用 HttpOnly、Secure Cookie
  - 设置合理过期时间
  - 登出时立即销毁服务端会话
  - 限制并发会话数
- **登录保护**：
  - 实施速率限制和账户锁定
  - 使用验证码防自动化攻击
  - 统一错误信息（不区分用户不存在和密码错误）
- **密码重置安全**：
  - 令牌随机生成且一次性使用
  - 设置短有效期（通常 15 分钟）
  - 重置后通知用户

## NIST 密码指南 (SP 800-63B)

- 最小长度 8 字符（推荐更長）
- 支持所有打印字符和空格
- 不强制定期更换密码（除非有泄露迹象）
- 不使用密码提示
- 验证新密码是否出现在已知泄露密码库中
